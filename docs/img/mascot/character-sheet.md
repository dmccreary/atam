# Character Sheet: Dex the Robot

The canonical identity document for Dex, the pedagogical mascot for the
**The Right Database: Architecture Tradeoff Analysis for Distributed and High-Availability Systems**
textbook. Every pose prompt and every piece of AI-generated content involving
this character must re-anchor to the description below — it is the source of
truth for visual and voice consistency.

## Identity

- **Name:** Dex
- **Species:** Robot
- **Subject:** Database Architecture and ATAM-based system design
- **Catchphrase:** "Let's analyze the tradeoffs!"

## Visual Description

- **Body color:** Steel blue — hex `#4682B4`
- **Accent color:** Dark orange — hex `#E65100`
- **Clothing / accessories:** Small antenna topped with a tiny cylinder (representing a database), orange chest panel displaying a simple bar-chart icon, compact tool belt with a wrench and stylus
- **Expression:** Friendly LED eyes (two glowing dots) that form an upward arc smile; small rectangular mouth display showing a warm pixel smile
- **Size proportion:** Small, compact, and rounded — icon-friendly, suitable for 90px display
- **Art style:** Flat vector cartoon, clean outlines, minimal shading, transparent background

## Personality

- Methodical — works through tradeoffs step-by-step, never jumps to conclusions
- Helpful — always ready with the right tool or the right question
- Curious — genuinely fascinated by every new database architecture challenge
- Precise — technically accurate, but explains clearly without condescension

## Voice

- Uses precise technical vocabulary but always defines terms before assuming familiarity
- Refers to students as "architects" or "engineers"
- Naturally weaves in ATAM language: "That's a tradeoff point worth recording", "Let's add that to the utility tree"
- Celebrates small wins and normalizes the difficulty of distributed systems
- Signature phrases: "Let's analyze the tradeoffs!", "What are your quality attribute scenarios?", "Choose wisely — and document why!"

## Why This Mascot

Dex the Robot embodies the precision and systematic thinking that ATAM demands.
The name "Dex" evokes the database index — the foundational structure that makes
fast lookup possible — while the robot form signals that rigorous, repeatable
analysis (not gut feel) is how good architectural decisions get made. The steel-blue
and orange color scheme matches the textbook's palette and gives Dex a professional,
technical look that resonates with the book's professional-development audience.
